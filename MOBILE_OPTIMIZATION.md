# Оптимизация для мобильных устройств 📱

## Что было сделано

### 1. **Улучшенные размеры кнопок (Touch-friendly)**
- Минимальная высота кнопок: **44px** (стандарт iOS)
- Минимальная высота input/select: **44px**
- Правильные отступы для удобного нажатия на сенсорном экране

```css
.btn {
    min-height: 48px;
    padding: var(--space-md) var(--space-xl);
    display: inline-flex;
    align-items: center;
    justify-content: center;
}

input, textarea, select {
    min-height: 44px;
    padding: 0.75rem;
    font-size: 16px; /* Предотвращает зум на iOS */
}
```

### 2. **Оптимизированный Hamburger меню**
- Размер кнопки: 44x44px (легче нажимать)
- Толще линии: 2.5px вместо 2px (лучше видно на мобильном)
- Лучше центрирование иконки

```css
.hamburger {
    min-width: 44px;
    min-height: 44px;
    align-items: center;
    justify-content: center;
}

.hamburger span {
    width: 26px;
    height: 2.5px;
}
```

### 3. **Адаптивные размеры шрифтов**
На мобильных устройствах (макс. 768px):
- Заголовок H1: 1.75rem (вместо 2rem) - более читаемо
- Заголовок H2: 1.5rem
- Заголовок H3: 1.25rem
- Основной текст: 0.9rem

### 4. **Улучшенная сетка курсов**
На мобильных:
- **1 столбец** вместо 3 на десктопе
- Правильное расстояние между картами: 1rem
- Полная ширина карт для лучшей видимости

```css
.courses-grid {
    grid-template-columns: 1fr; /* Один столбец на мобильном */
    gap: 1rem;
    padding: 0 0.75rem;
}
```

### 5. **Оптимизированная страница урока**
- Боковая панель переходит в положение `static` на мобильном
- Правильный размер блоков кода: 0.8rem шрифт
- Включена прокрутка для длинных кодов: `-webkit-overflow-scrolling: touch`

```css
.code-block {
    font-size: 0.8rem;
    overflow-x: auto;
    -webkit-overflow-scrolling: touch; /* Гладкая прокрутка на iOS */
}
```

### 6. **Улучшенная навигация**
- Sticky navbar в верхней части экрана
- Фиксированная позиция для быстрого доступа к меню
- Правильное расстояние между ссылками в меню

### 7. **Форма элементы**
- Минимум 44px высота для всех input полей
- Шрифт 16px (предотвращает нежелательный зум на iOS)
- Правильный фокус стиль: золотая граница с тенью

```css
input:focus, textarea:focus, select:focus {
    outline: none;
    border-color: var(--accent-primary);
    box-shadow: 0 0 0 3px rgba(212, 175, 55, 0.1);
}
```

### 8. **Оптимизация для ланшафтной ориентации**
- Меньше паддинг в ланшафтном режиме
- Более компактное расположение элементов

### 9. **Touch device оптимизации**
Для сенсорных экранов добавлены:
- Эффект масштабирования при нажатии (scale 0.98)
- Активные состояния кнопок более заметны
- Отладка hover состояний (скрыты на touch устройствах)

## Точки разлома (Breakpoints)

| Устройство | Максимальная ширина | CSS правило |
|-----------|-------------------|-----------|
| Мобильный (маленький) | 480px | `@media (max-width: 480px)` |
| Мобильный (больший) | 768px | `@media (max-width: 768px)` |
| Планшет | 1024px | `@media (max-width: 1024px)` |
| Десктоп | 1200px+ | По умолчанию |

## Тестирование на мобильном

### В браузере DevTools:
1. Откройте **F12** (Developer Tools)
2. Нажмите **Ctrl+Shift+M** (или иконка мобильного телефона)
3. Выберите размер экрана:
   - iPhone 12: 390x844px
   - iPhone 14: 393x852px
   - Samsung Galaxy S20: 360x800px
   - iPad: 768x1024px

### На реальном мобильном:
1. Откройте в браузере: `http://192.168.x.x:8000` (ваш IP адрес)
2. Проверьте:
   - ✅ Все кнопки нажимаются легко (44px минимум)
   - ✅ Текст читаемый без зума
   - ✅ Нет горизонтальной прокрутки
   - ✅ Меню Hamburger работает
   - ✅ Картинки загружаются правильно
   - ✅ Форма работает (если есть)
