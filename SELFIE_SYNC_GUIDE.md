# 📱 Руководство синхронизации селфи

## 🎯 Как это работает

1. **Вход на сайт** — При открытии сайта на новом устройстве:
   - `selfi.html` захватывает скриншот (через html2canvas)
   - Скриншот отправляется на сервер
   - Скриншот сохраняется в локальное хранилище (localStorage)

2. **Синхронизация между устройствами**:
   - Все селфи хранятся на сервере в `data/gallery-selfies.json`
   - При открытии галереи (`gallery.html`) загружаются селфи с сервера
   - Если сервер недоступен — используется кэш из localStorage

3. **Удаление селфи**:
   - Удаление работает со всех устройств
   - Сначала удаляет с сервера, затем обновляет localStorage

## 🔧 Как работает синхронизация

### GET /api/selfies
```javascript
// Загрузить все селфи
fetch('/api/selfies')
  .then(r => r.json())
  .then(selfies => {
    // selfi.id, photo, timestamp, device
  })
```

**Порядок загрузки:**
1. Сначала пробует загрузить с сервера
2. Если сервер недоступен → используется localStorage кэш
3. Если кэша нет → показывает пустую галерею

### POST /api/selfies
```javascript
// Сохранить новое селфи
fetch('/api/selfies', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    photo: base64ImageData,
    device: 'Android|iOS|Desktop'
  })
})
```

**После сохранения:**
1. Сохраняет на сервер → `data/gallery-selfies.json`
2. Кэширует в localStorage → `gallery_selfies_cache`
3. Если сервер недоступен → сохраняет только в localStorage

### DELETE /api/selfies/:id
```javascript
// Удалить селфи по ID
fetch(`/api/selfies/${id}`, { method: 'DELETE' })
```

**Порядок удаления:**
1. Удаляет с сервера
2. Обновляет localStorage кэш
3. Все браузеры автоматически видят изменение (если галерея открыта)

## 📊 Данные селфи

```javascript
{
  "id": "1710412200000xyz123",
  "photo": "data:image/jpeg;base64,...",  // Base64 скриншот
  "timestamp": "2024-03-14T10:30:00.000Z", // ISO время
  "timeString": "14.03.2024, 10:30:00",   // Локальный формат
  "device": "Android|iOS|Desktop"          // Откуда было сохранено

```

**Размер:**
- Одно селфи: ~100-300 KB (скриншот JPEG 0.85 качества)
- Максимум селфи: 100 последних (старые удаляются автоматически)
- Максимум данных на сервер: 5 MB за один запрос

## ⚠️ Проблемы и решения

### Проблема: Селфи не синхронизируются между устройствами

**Решение:**
1. Проверьте, что оба устройства подключены к одной сети
2. Используйте IP адрес вместо `localhost` (например: http://192.168.1.100:8000)
3. Откройте консоль (F12) и проверьте ошибки в сети
4. Убедитесь, что порт 8000 не заблокирован фаерволом

### Проблема: Селфи исчезает после перезагрузки

**Это нормально если:**
- Браузер очищает localStorage при закрытии
- Сервер остановлен и нет fallback

**Решение:**
- Отключите режим приватного/инкогнито браузера
- Запустите сервер: `node server.js`

### Проблема: Большой размер селфи занимает место

**Решение:**
1. Уменьшите масштаб скриншота (измените `scale: 0.5` на `0.3-0.4`)
2. Уменьшите качество JPEG (измените `0.85` на `0.7`)
3. Удалите старые селфи через галерею

## 🔐 Безопасность

- Галерея защищена паролем: `пароль123`
- Попытка 5 раз с неправильным паролем → блокировка на 60 секунд
- Селфи хранятся в Base64 (текстовой формат) для легкости синхронизации
- Нет шифрования (передается по HTTP, используйте HTTPS на продакшене)

## 📝 Файлы системы

```
selfi.html                  ← Вход, захват скриншота, отправка на сервер
gallery.html                ← Админ-галерея, просмотр и удаление
server.js                   ← Node.js сервер с API
data/gallery-selfies.json   ← Хранилище селфи на сервере
```
