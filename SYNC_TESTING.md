# 📱 Инструкция по тестированию синхронизации селфи между устройствами

## 🚨 ВАЖНО: Используйте правильный IP адрес!

**Проблема:** На телефоне не видны селфи с ноутбука?

**Причина:** Если вы открываете `http://localhost:8000` на телефоне, это НЕ будет работать!
- `localhost` = только ваш компьютер
- Телефон в другой сети не может подключиться

**Решение:** Используйте IP адрес ноутбука вместо `localhost`

---

## 🔍 Шаг 1: Найти IP адрес ноутбука

### На Windows (PowerShell):

```powershell
ipconfig
```

Найдите строку `IPv4 Address: 192.168.x.x` (обычно в сети wifi)

Пример вывода:
```
Ethernet adapter:
   IPv4 Address . . . . . . . . . . . : 192.168.1.100

Это и есть ваш IP адрес!
```

### На Mac/Linux:

```bash
ifconfig | grep "inet "
```

Ищите строку типа `192.168.1.100` или `10.0.0.x`

## 🐛 Решение проблем

### Проблема: На телефоне не видны селфи

**Проверьте:**
1. ✅ Сервер запущен на ноутбуке (`node server.js`)
2. ✅ Используете правильный IP адрес (не localhost)
3. ✅ Телефон в одной WiFi сети с ноутбуком
4. ✅ Открыли F12 консоль на телефоне
5. ✅ Нажали "Продолжить" на ноутбуке (селфи должно быть сохранено)

**Как отладить:**
- На ноутбуке: смотрите логи в консоли Node.js
- На телефоне: смотрите логи F12 console в браузере
- Ищите ошибки вроде `❌` или `⚠️`

### Проблема: Ошибка "Cannot GET /api/selfies"

**Это значит:** Сервер на Node.js не запущен!

**Решение:**
```bash
cd "c:\Users\ххх\Desktop\SCHOOL9"
node server.js 8000
```

### Проблема: "Failed to fetch" в консоли браузера

**Возможные причины:**
1. Неправильный IP адрес → проверьте `ipconfig`
2. Телефон в другой WiFi сети → подключитесь к одной сети
3. Порт 8000 заблокирован фаерволом → отключите firewall или разрешите порт
4. Сервер не запущен → запустите `node server.js`

### Проблема: "Error: ENOENT: no such file or directory, mkdir"

**Это нормально!** Сервер автоматически создает папку `data/`

Просто игнорируйте и продолжайте
